Upgrade concept pages from link indexes into evidence-grounded expert pages.

Argument: $ARGUMENTS (optional concept slug such as `diffusion-model`; omit to process all `wiki/concepts/*.md`)

Purpose:
Concept pages are not finished just because `Related Papers` is populated. A finished concept page must help answer domain questions, compare methods, and navigate the literature graph.

Workflow:
1. Select target concept pages:
   - If $ARGUMENTS is provided, process only `wiki/concepts/$ARGUMENTS.md`.
   - Otherwise process all files under `wiki/concepts/`.
2. First run the deterministic local-evidence upgrader:
```powershell
python .claude/scripts/upgrade_concept_pages.py $ARGUMENTS
```
3. Then manually inspect any changed concept page that matters for the user's task. For each concept page, read:
   - The concept page itself.
   - Its `Related Papers` links.
   - At least the top 8 related literature notes, prioritizing `status: done`, recent papers, and seminal/classic papers.
4. If the deterministic draft is too shallow, refine the concept page using local evidence only. Preserve frontmatter keys and aliases, but set:
   - `status: done` only after the sections below are filled.
   - `updated` to today.
5. Required concept sections:
   - `## Definition`: concise local-domain definition.
   - `## Key Ideas`: 5-8 bullets that synthesize recurring ideas across local papers.
   - `## Method Families`: grouped methods with representative local paper links.
   - `## Key Papers`: 5-12 local papers, each with one concrete reason it matters.
   - `## Evidence Map`: what the vault currently supports, including strongest evidence and gaps.
   - `## Open Problems`: unresolved issues, failure modes, and research opportunities.
   - `## Related Concepts`: bidirectional concept links where appropriate.
   - `## Related Papers`: keep the complete reverse-link list generated by `backfill_concepts.py`.
6. Do not invent external history or unverified claims. If the vault lacks evidence for a subtopic, write that as a local evidence gap.
7. After editing concept pages, run:
```powershell
python .claude/scripts/backfill_concepts.py
python .claude/scripts/audit_kb.py --strict-reading --strict-concepts
```
8. If strict concept audit fails, fix the reported concept page and rerun the audit.

Completion report:
- Concepts processed.
- Concepts changed.
- Any concepts still `status: stub`.
- `audit_kb.py --strict-reading --strict-concepts` result.
